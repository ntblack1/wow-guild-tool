# 后续开发计划

## 当前阶段

- React + Vite + TypeScript 项目骨架。
- Tailwind 轻奇幻设计系统。
- 首页、活动、论坛、角色、战报页面。
- Supabase Auth/Database 基础连接。
- 角色、活动、报名、发帖、评论基础闭环。
- 活动报名第一版可用闭环：
  - 所有已登录成员都可以发起活动。
  - 活动列表显示人数、上限、状态和阵容缺口。
  - 活动详情显示当前用户报名状态。
  - 登录用户可选择角色报名或取消报名。
  - 同一角色重复报名有友好提示。
  - 阵容按 T / 治疗 / DPS 分组。
  - 管理员/团长可修改报名状态。
  - 首页与活动页优先展示今日活动，报名流程为选择角色后一次确认。
  - 角色支持上传自定义头像，阵容列表展示头像。
- 工会论坛第一版可用闭环：
  - 发帖、帖子详情、评论。
  - 板块筛选。
  - 最新/热门排序，热门暂按评论数排序。
  - 置顶帖优先展示。
  - 管理员/团长可置顶或取消置顶。
- Supabase SQL 和 RLS：
  - 核心表：profiles、characters、events、signups、posts、comments。
  - 保留 reports 作为战报原型表。
  - 匿名可读活动、报名阵容、帖子和评论。
  - 登录后可写自己的角色、报名、帖子和评论。
  - 用户可删除自己的报名，管理员/团长也可删除。
  - 账号密码由 Supabase Auth 管理，不在业务表中保存密码摘要。
  - 普通成员只能修改自己的数据；活动管理、报名状态和帖子置顶仅限管理员/团长。

## 下一阶段

- 将战报页接入 `reports` 表真实 CRUD。
- 补充公告表，或将公告作为论坛置顶板块管理。
- 给活动增加更明确的缺口字段和报名人数聚合，减少列表页多次查询。
- 给帖子增加更完整的热度依据，例如浏览量或点赞。
- 增加微信分享图和 favicon。
- 增加更完整的表单校验。
