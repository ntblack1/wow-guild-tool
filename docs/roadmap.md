# 后续开发计划

## 当前阶段

- React + Vite + TypeScript 项目骨架。
- Tailwind 轻奇幻设计系统。
- 首页、活动、论坛、角色、战报页面。
- 首页首屏优先展示今日/下一场活动，并在活动卡片提示当前成员的报名状态。
- 首页公告与黑红榜接入真实帖子和战报，各数据区独立容错。
- Supabase Auth/Database 基础连接。
- 角色、活动、报名、发帖、评论基础闭环。
- 角色页对未登录成员显示清晰登录入口，登录后再展示管理表单。
- 角色、活动和论坛登录入口保留安全的站内返回地址，登录后可继续原操作。
- 活动报名第一版可用闭环：
  - 所有已登录成员都可以发起活动。
  - 发布成功后直接进入新活动详情；取消报名采用轻量二次确认，避免误触。
  - 活动列表显示人数、上限、状态和阵容缺口。
  - 活动入口文案按状态与个人报名自动变化，活动筛选写入网址并在详情返回时保留。
  - 活动详情显示当前用户报名状态。
  - 登录用户可选择角色、本次职责和可选备注报名，也可取消报名。
  - 同一账号在同一活动只占一个席位，重复报名有友好提示。
  - 报名插入策略只允许普通成员创建“已报名”或“替补”，确认状态只能由团长/管理员更新。
  - 阵容按 T / 治疗 / DPS 分组。
  - 管理员/团长可修改报名状态。
  - 活动发起人可编辑活动信息并开放、锁定或结束自己的活动。
  - 编辑人数上限时校验当前正式阵容，不能把容量降低到已报名人数以下。
  - 首页与活动页优先展示今日活动，报名流程为选择角色后一次确认。
  - 发起活动表单按需展开，活动列表提供“我的报名”筛选并限制查询数量。
  - 角色支持上传自定义头像、即时图片校验和大尺寸裁切预览，阵容列表展示头像。
  - 删除角色采用二次确认，并提示会同步移除该角色的活动报名。
- 工会论坛第一版可用闭环：
  - 发帖、帖子详情、评论。
  - 板块筛选。
  - 最新/热门排序，热门暂按评论数排序。
  - 置顶帖优先展示。
  - 管理员/团长可置顶或取消置顶。
  - 评论支持回复、引用与作者确认删除，长帖默认加载最新 20 条并可按需加载更早内容。
  - 发帖人可编辑自己的未置顶帖子、删除自己的帖子，详情页有返回和面包屑导航。
  - 论坛首屏优先展示帖子流，发帖编辑器按需展开，减少手机端首屏滚动。
  - 论坛支持服务端标题搜索；搜索、板块与排序状态写入网址，进入帖子后返回仍保留原视图；帖子列表每次只追加 20 条，避免一次加载过多。
  - 评论按父子关系缩进排列，默认加载最新 20 条并支持按需加载更早内容。
  - 成员可上传、调整裁切位置或撤下限量压缩的共享展示图，成员展示自动滚动。
  - 首页和活动页按实际显示范围查询，活动主体先显示、阵容随后补齐且加载时不误报 0 人；“我的报名”会等待阵容数据后再显示结果；非首页路由按需加载；论坛、战报和成员展示先显示公共内容，权限资料随后补齐；公共依赖拆包以提高浏览器缓存命中率。
  - 用户可见错误统一转换为中文行动提示，数据库约束、权限策略和英文错误码不直接暴露给成员。
  - 主要数据页的错误状态支持原地重新读取，恢复操作不会自动重放写入请求。
  - 论坛作者、评论作者和成员展示查询遵循最小字段原则，避免传输页面未使用的账号资料。
  - 新帖编辑器支持 7 天本机草稿，采用延迟写入并在发布成功后清除，不新增数据库请求。
  - 活动详情可复制适合微信群发布的分组阵容文本，包含活动信息、报名状态、备注和链接。
  - Cloudflare Pages 包含显式 SPA 回退规则，微信群直达活动或帖子详情不会落入静态 404。
  - 生产构建显式启用 React 生产模式并提前采用 Router v7 兼容开关，减少脚本体积和控制台警告。
  - 全局错误边界避免异常白屏，浏览器离线时显示不遮挡导航的网络状态提示。
  - 正式报名阵容不计替补与请假，名额满后新报名自动进入替补并单独展示。
  - 活动卡片和详情显示开团发起人，创建者昵称与活动在同一次查询中返回。
  - 活动与帖子详情支持系统分享，无法调起分享面板时自动复制链接；网站包含 favicon、页面标题和群聊预览元信息。
  - 首页和活动列表只读取今天及之后的未结束活动，过期活动在次日自动隐藏但不删除历史记录。
- Supabase SQL 和 RLS：
  - 核心表：profiles、characters、events、signups、posts、comments。
  - 战报读取真实 `reports` 数据，团长/管理员可发布、编辑和删除。
  - 最新红手、黑手和金句自动取最近战报，历史列表限制 12 条并按需展开长正文。
  - 匿名可读活动、报名阵容、帖子和评论。
  - 登录后可写自己的角色、报名、帖子和评论。
  - 用户可删除自己的报名，管理员/团长也可删除。
  - 报名唯一约束迁移会先确定性清理历史重复记录，避免已有测试数据阻断 SQL 执行。
  - 账号密码由 Supabase Auth 管理，不在业务表中保存密码摘要。
  - 登录后的账号页可修改工会昵称，并同步到导航和论坛作者资料。
  - 账号中心显示普通成员、团长或管理员身份与权限范围，角色提升仍只允许数据库管理员执行。
  - 账号中心显示短登录账号，成员可在登录状态下两次确认后修改自己的密码。
  - 普通成员只能修改自己的数据；活动管理、报名状态和帖子置顶仅限管理员/团长。
