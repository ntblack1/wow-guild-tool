# 八块腹肌工会 Web MVP 设计

## 背景

项目从原“小程序 + Express + Prisma”方向调整为普通网页应用。第一阶段不做微信小程序上线审核，不开发复杂后台，使用微信群分享链接访问。

## 目标

创建一个可运行、可部署、可连接 Supabase 的最小可用 Web 应用：

- React + Vite + TypeScript + Tailwind CSS
- Supabase 负责登录、数据库与行级权限
- Cloudflare Pages 部署
- GitHub 托管
- 手机竖屏优先，深色工会大厅风格，金色标题，卡片式布局

## 范围

第一版包含以下页面：

- 首页：工会名称、最近活动、最新帖子、活动报名/工会论坛/我的角色入口
- 登录页：邮箱登录/注册的轻量入口
- 我的角色：创建、查看、更新、删除自己的角色
- 活动列表：查看活动，管理员可创建活动
- 活动详情：查看活动信息，选择自己的角色报名，按 T/N/DPS 展示阵容
- 论坛列表：查看帖子，创建帖子
- 帖子详情：查看正文和评论，发表评论

第一版不包含：

- 微信登录
- 支付
- 小程序目录改造
- 复杂权限后台
- 排行榜
- 官方素材或可能侵权资源
- 活动海报

## 数据模型

Supabase SQL 创建以下表：

- `profiles`：用户资料和简单角色权限，字段包括 `id`、`display_name`、`role`、时间戳
- `characters`：用户角色，字段包括角色名、职业、天赋、职责、装等、备注
- `events`：活动，字段包括标题、副本、开团时间、人数上限、说明、状态、创建者
- `signups`：报名，字段包括活动、角色、职责、备注、报名状态
- `posts`：论坛帖子，字段包括标题、正文、板块、作者、置顶、创建时间
- `comments`：帖子评论，字段包括帖子、作者、正文、创建时间

权限规则使用 Supabase RLS：

- 登录用户可管理自己的 profile、角色、报名、帖子、评论
- 所有人可读取公开活动、报名阵容、帖子和评论
- `profiles.role in ('admin', 'leader')` 的用户可创建活动、更新报名状态、置顶帖子

## 架构

新增 `apps/web`：

- `src/lib/supabase.ts`：Supabase client
- `src/types.ts`：前端表类型和枚举字面量
- `src/App.tsx`：路由容器
- `src/pages/`：页面组件
- `src/components/`：导航、布局、卡片、表单等轻量组件
- `src/services/`：Supabase CRUD 封装

页面直接调用 `services`，不新增自建 Express API。这样部署最简单，也符合 Supabase + Cloudflare Pages 的目标。

## 数据流

1. 用户通过 Supabase Auth 登录。
2. 登录后前端读取 `profiles`，判断是否为普通成员、团长或管理员。
3. 角色、活动、报名、帖子、评论通过 Supabase client 读写。
4. 重要权限由 RLS 兜底，前端仅负责隐藏不该显示的入口。

## UI 方向

整体视觉是“非侵权的魔兽工会大厅感”：

- 深色背景，不使用官方图标、Logo、截图或纹理
- 金色标题和边框点缀
- 卡片式内容块，移动端单列优先
- 底部或顶部简洁导航，入口清晰
- 页面首屏直接展示应用，不做营销落地页

## 验证

每一步保持可运行：

- 项目骨架完成后可启动 Vite
- 路由完成后每个页面可打开
- Supabase 配置完成后缺少环境变量时显示友好提示
- CRUD 完成后运行构建和基础测试

最终至少运行：

- `npm run build`
- `npm run test`

如果网络限制导致依赖无法安装，需要请求授权后再安装依赖。
