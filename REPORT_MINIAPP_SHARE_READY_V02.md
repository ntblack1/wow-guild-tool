# REPORT_MINIAPP_SHARE_READY_V02

执行日期：2026-07-03

## 1. 目标

把当前项目推进到可以通过微信小程序给朋友手机测试的阶段。

## 2. 本次完成

- 新增运行时 API 地址配置工具：`apps/miniapp/utils/api-config.js`
- 小程序请求封装改为优先读取本地保存的 API 地址。
- 调试页新增：
  - 当前 API 地址显示
  - 默认 API 地址显示
  - 本地保存 API 地址显示
  - API 地址输入框
  - 保存 API 地址
  - 恢复默认 API 地址
  - 后端健康检查
- 新增小程序手机内测说明：`docs/dev/miniapp-share-testing.md`

## 3. 朋友手机测试必须满足

手机必须能访问后端。

不可用：

```text
http://127.0.0.1:3000
```

原因：手机里的 `127.0.0.1` 指的是手机自己，不是开发电脑。

可用于临时内测：

```text
http://电脑局域网IP:3000
```

更适合体验版/正式版：

```text
https://公网域名
```

## 4. 当前推荐测试步骤

1. 部署或启动一个手机能访问的后端。
2. 微信开发者工具上传体验版。
3. 微信公众平台添加体验成员。
4. 朋友打开体验版。
5. 进入“调试”页。
6. 填写后端 API 地址。
7. 点击“保存地址”。
8. 点击“测试后端健康检查”。
9. 健康检查成功后，进入“我的”页选择成员和新增角色。
10. 进入活动详情报名。

## 5. 仍需补齐

- 公网 HTTPS 后端部署。
- 微信公众平台 request 合法域名配置。
- 正式微信登录。
- 正式成员绑定。

## 6. 结论

代码侧已补到适合手机体验版调试的状态。

但如果要让朋友不在同一局域网也能测试，还必须先准备公网 HTTPS 后端地址。
