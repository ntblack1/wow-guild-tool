# wow-guild-tool miniapp

微信小程序基础版 V0.1，使用原生微信小程序结构。

## 打开方式

1. 在项目根目录准备数据库和测试数据：

```bash
npm run db:migrate
npm run db:seed
```

2. 启动后端：

```bash
npm run dev
```

3. 使用微信开发者工具打开 `apps/miniapp`，或在项目根目录打开根 `project.config.json`。

如果没有真实 AppID，可以使用测试号或本项目配置中的 AppID。本地开发已关闭 URL 合法域名校验，方便访问本机后端。

## 本地端口说明

- 开发版后端 API 默认地址：`http://192.168.50.101:3000`
- 小程序配置文件：`apps/miniapp/config/index.js`
- 微信开发者工具“设置 - 安全设置 - 服务端口”的端口，例如 `25404`，是开发者工具自动化/服务端口，不是本项目后端 API 端口。

调试页健康检查请求的是 `API_BASE_URL + /health`，所以本地联调时应确认 `API_BASE_URL` 指向后端 Express 服务。

## 本地配置

配置文件：`apps/miniapp/config/index.js`

- `MINIAPP_API_BASE_URLS.develop`：默认 `http://192.168.50.101:3000`
- `MINIAPP_API_BASE_URLS.trial`：体验版 HTTPS 后端域名，上传体验版前必须填写
- `MINIAPP_API_BASE_URLS.release`：正式版 HTTPS 后端域名，正式发布前必须填写
- `CURRENT_MEMBER_ID`：默认 `seed_member_xiaoheiwa`
- `LEADER_ID`：默认 `seed_member_laodongrenmin`

上传体验版前在项目根目录运行：

```bash
npm run check:miniapp-release
```

内测阶段当前成员优先从微信小程序本地 storage 读取，storage key 为 `wow_guild_tool_current_member_id`。`CURRENT_MEMBER_ID` 只作为兜底默认值，不是正式登录身份。

## 页面

- 首页：活动列表
- 活动详情：活动信息、报名统计、报名入口、报名名单入口
- 报名页：角色、职责、备注、提交报名
- 我的：成员信息、当前成员切换、角色新增/编辑/删除、设置主号、报名记录
- 团长管理：创建活动、开放报名、锁定报名、查看报名、生成签到、查看签到、修改出勤状态、查看出勤统计
- 调试：健康检查、配置、测试数据说明

## 推荐演示路径

1. 调试页检查后端健康状态。
2. 我的页选择当前成员，例如“小黑娃”。
3. 我的页检查角色列表；如没有角色，点击“新增角色”维护角色。
4. 首页进入“八块腹肌今晚开团”。
5. 活动详情页点击报名。
6. 使用当前成员的角色报名。
7. 返回活动详情页查看报名统计变化。
8. 团长管理页查看报名名单。
9. 团长锁定报名。
10. 团长生成签到并查看签到名单。
11. 将成员改为“迟到”，查看出勤统计。

## 内测成员切换说明

当前成员切换只是内测方案，用于替代微信登录前的身份选择：

1. 打开“我的”页。
2. 在“切换当前成员”里选择成员。
3. 选择结果保存到本地 storage。
4. 报名页、我的页会优先使用本地选择的 memberId。
5. “调试”页和“我的”页都可以清除当前成员。

后续接入微信登录时，应由后端根据微信 openid 解析当前成员，并替换 `utils/current-member.js` 中的本地 storage 读取逻辑。前端页面继续调用同一个当前成员工具，避免散落改动。

## 我的角色管理

成员第一次使用时按这个顺序操作：

1. 打开“我的”页。
2. 在“切换当前成员”中选择自己。
3. 在“我的角色”中点击“新增角色”。
4. 填写角色名、职业、专精、职责、装等，并按需设为主号。
5. 保存后角色列表会刷新。
6. 回到活动详情页报名，报名页会读取当前成员的角色列表。

角色管理支持：

- 新增角色
- 编辑角色
- 设置主号
- 删除角色，删除前会二次确认

同一个成员最多只有一个主号；设置新主号后，原主号会自动取消。
