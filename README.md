# 八块腹肌工会 Web 工具

给魔兽世界“八块腹肌工会”使用的活动报名、工会论坛与战报 Web 应用。第一版采用普通网页形式，通过微信群分享链接使用，不做微信小程序上线审核。

视觉方向：轻奇幻、移动端优先，不使用任何魔兽世界官方素材。

## 技术栈

- React + Vite + TypeScript
- Tailwind CSS
- lucide-react
- Supabase Auth + Database
- Cloudflare Pages

## 目录

```text
apps/web/             Web 前端应用
supabase/schema.sql   Supabase 建表和 RLS SQL
docs/design-system.md 设计系统
docs/design-assets.md 视觉素材说明
docs/roadmap.md       后续开发计划
```

旧的 `apps/miniapp` 和 `apps/server` 暂时保留；当前网页 MVP 的主要代码在 `apps/web`。

## 本地运行

安装依赖：

```bash
npm install
```

复制环境变量：

```powershell
Copy-Item .env.example .env
```

在 `.env` 中填写：

```bash
VITE_SUPABASE_URL="https://your-project.supabase.co"
VITE_SUPABASE_ANON_KEY="your-public-anon-key"
```

这两个值必须来自你的 Supabase 项目。修改 `.env` 后需要重新启动 Web 开发服务，页面才会读到新配置。

启动 Web 开发服务：

```bash
npm run dev:web
```

构建：

```bash
npm run build:web
```

测试：

```bash
npm run test:web
```

## Supabase 配置

1. 在 Supabase 创建新项目。
2. 打开 Supabase SQL Editor。
3. 执行 `supabase/schema.sql` 中的全部 SQL。
4. 打开 Authentication > Providers > Email，关闭 `Confirm email`（确认邮箱）。
5. 在 Project Settings 中找到 Project URL 和 anon public key。
6. 填入本地 `.env` 和 Cloudflare Pages 环境变量。

第一版登录使用工会口令 + 账号密码。成员先回答“会长口头禅是什么”，输入 `说这些` 后即可创建账号和密码；之后直接用账号密码登录。成员不需要填写真实邮箱，程序会在内部生成专用登录地址，并由 Supabase Auth 安全保存密码和登录会话。

如果之前已经执行过旧版 SQL，也可以重新执行当前 `supabase/schema.sql`。脚本会修正中文约束、删除旧的自建密码表，并把写入权限绑定到 Supabase 登录身份。旧测试版创建的本地账号不能继续使用，需要重新创建一次账号。

启用角色头像时，额外在 SQL Editor 执行 `supabase/migrations/20260715_add_character_avatars.sql`。它会添加头像字段并创建一个仅允许成员管理自己文件夹的公开头像存储桶。

账号只能使用 3-20 位小写字母、数字和下划线；工会昵称可以使用中文。口令是第一版内部测试的注册门槛，不应当作高强度安全措施。

### 设置管理员

用户第一次通过工会口令创建账号后，Supabase 会自动写入 `profiles`。在 SQL Editor 中先查看用户：

```sql
select id, display_name, role from public.profiles order by created_at desc;
```

把某个用户设为管理员：

```sql
update public.profiles
set role = 'admin', display_name = '团长昵称'
where id = '用户 uuid';
```

团长也可以使用：

```sql
update public.profiles
set role = 'leader'
where id = '用户 uuid';
```

成员重新进入账号中心后，会看到“普通成员 / 团长 / 管理员”身份和对应权限说明。账号身份只能通过 Supabase SQL 设置，普通成员不能自行提升权限。

账号中心同时显示登录账号，并允许已登录成员两次确认后修改自己的密码；内部认证邮箱不会展示给成员。

## Cloudflare Pages 部署

1. 将代码推送到 GitHub。
2. 在 Cloudflare Pages 新建项目并连接仓库。
3. 构建设置：
   - Framework preset: Vite
   - Root directory: `/`
   - Build command: `npm run build:web`
   - Build output directory: `apps/web/dist`
4. 在 Cloudflare Pages 的 Environment variables 中添加：
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
5. 部署后，把 Pages 生成的网址分享到微信群。

`apps/web/public/_redirects` 会随构建发布，把活动详情、帖子详情等直接链接交给 React 路由处理，避免成员从微信群打开深层链接时出现 404。

`apps/web/public/_headers` 会随构建发布，阻止页面被第三方网站嵌入，关闭未使用的摄像头、麦克风、定位和支付权限，并让带哈希的静态资源长期缓存。

## 当前 MVP 功能

- 首页首屏优先展示今日/下一场活动和常用入口，再展示紧凑的工会大厅视觉、成员图、最近活动与最新帖子。
- 首页主活动不会在下方重复出现；活动页默认将“今日活动”和“后续活动”分开，切换筛选时只显示对应结果。
- 首页公告读取置顶帖或开团通知，黑红榜读取最新战报，不再显示固定假数据。
- 首页和活动列表使用一次限量查询同时读取活动与对应阵容，减少慢网络下的一次 Supabase 往返；卡片只读取实际展示的角色字段，不会用“0 人”代替加载状态。
- 登录、活动、角色、论坛、战报和图片操作统一显示简短中文错误，不向成员暴露数据库表名、权限策略或英文错误码。
- 活动、论坛、帖子、角色和战报的数据错误提供原地“重新读取”，只刷新当前数据，不会重复提交报名、帖子或删除操作。
- 页面数据读取期间使用稳定的内容骨架，内嵌小区域使用紧凑提示；不增加数据库请求，也避免慢网络下出现近似空白页。
- 论坛作者、评论作者和成员展示使用明确字段清单，不会为列表页面下载完整账号资料或无关图片字段。
- 发帖内容在停止输入约半秒后保存为当前设备草稿，7 天内可自动恢复，发布成功或手动清空后删除；草稿不会上传数据库。发布成功后直接进入新帖详情，并保留返回论坛时的板块、排序和搜索条件。
- 活动详情支持一键复制微信群阵容文本，自动按坦克、治疗、输出、替补和请假分组，并包含状态、备注与活动链接。
- 工会口令创建账号，之后使用账号密码登录；登录、注册和修改密码表单支持浏览器密码管理器，并可临时显示密码核对输入。
- 密码和登录会话由 Supabase Auth 管理，数据库写入受登录身份和角色权限保护。
- 登录后的账号页可修改工会昵称，并直接查看自己的下一场活动、出战角色和报名状态；登录、退出或修改昵称后，顶部账号入口会立即同步，无需刷新网页。
- 从角色、活动或论坛进入登录时，登录或创建账号成功后会安全返回原页面，继续刚才的操作。
- 用户创建、编辑、删除自己的多个角色；删除采用二次确认，并明确提示会移除相关报名。
- 未登录进入角色页时只显示明确的登录入口，登录后才显示角色表单，避免新成员误以为禁用表单发生故障。
- 角色可上传自定义头像，选图时立即检查大小，并通过大预览调整头像裁切位置。
- 所有已登录成员都可以发起活动，活动默认进入报名中。
- 活动发布成功后直接进入新活动详情，方便立即查看阵容和分享；取消报名和手动结束活动都需要明确确认，避免误触丢失席位或让活动提前消失。
- 发起活动表单默认收起，手机首屏优先显示今日活动和活动列表。
- 活动发起人可以修改自己活动的名称、时间、人数和说明，并开放、锁定或结束活动。
- 修改人数上限时不能低于当前正式报名人数，避免已有阵容与活动容量互相矛盾。
- 发起活动支持常用副本预设和自定义名称，标题自动生成，说明为选填。
- 活动列表显示报名人数、人数上限、状态和当前阵容缺口。
- 活动卡片会按状态显示“立即报名、查看我的报名、查看当前阵容或查看活动结果”，活动筛选在进入详情后仍会保留。
- 活动卡片和详情显示开团发起人，所有活动查询一次带回昵称，不增加额外请求。
- 活动列表只显示今天及之后的未结束活动；活动日期过去后会在第二天自动隐藏，但数据库记录和原详情链接仍然保留。登录成员可用“我的报名”快速筛选。
- 首页和活动页优先展示今日活动；报名页选择角色和本次职责后即可确认，备注保持选填。
- 活动卡片、活动详情、个人下一场和微信群阵容文本统一显示星期，减少成员看错开团日期。
- 首页活动卡片显示真实报名人数、报名角色头像与角色名，头像旁的职责统一显示为 T、治疗或 DPS。
- 已登录成员会在活动卡片上直接看到自己的报名角色和状态。
- 成员选择自己的角色报名，已报名后可以取消报名。
- 报名中的活动允许成员直接更换自己的角色、职责和备注，原有确认/替补状态保持不变，不需要先取消再重新排队。
- 每个账号在同一活动只占一个席位，重复报名会显示友好提示。
- 数据库权限限制普通成员的新报名只能进入“已报名”或“替补”，不能绕过网页自行伪造“已确认”。
- 活动详情按 T / 治疗 / DPS 展示正式阵容，替补和请假单独统计；正式名额满后新报名自动进入替补。
- 活动和帖子详情支持一键分享；不支持系统分享的浏览器会自动复制当前链接，方便粘贴到微信群。
- 活动详情可生成标准日历文件，包含开团时间、详情链接和提前 30 分钟提醒，便于成员加入手机日历。
- 网站包含独立工会图标、中文页面标题和群聊分享预览信息，预览图使用项目原创大厅图片。
- 网站提供移动端 Web App 清单，可从手机浏览器添加到主屏幕，并可从快捷入口直达活动、论坛和角色。
- 管理员/团长修改报名状态：已报名、已确认、替补、请假。
- 用户发帖、筛选板块、按最新/热门浏览帖子；移动端默认优先显示帖子流，按“发布新帖”后再展开编辑器。
- 论坛支持按标题搜索；搜索、筛选与排序会保留在网址中，查看帖子后返回仍保持原来的板块视图；帖子列表每次只追加 20 条。
- 评论支持回复、引用主楼或评论，并允许作者确认后删除自己的评论；长帖默认加载最新 20 条，可继续加载更早内容。
- 发帖人可以编辑自己的未置顶帖子或删除自己的帖子，详情页提供明确的返回与层级导航。
- 帖子置顶优先展示，管理员/团长可以置顶或取消置顶。
- 帖子详情展示正文、作者和分层回复；默认只加载最新 20 条，可按需加载更早评论，登录用户可以评论。
- 战报页读取真实数据，展示最新红手、黑手和金句；团长/管理员可发布、编辑、二次确认删除战报。
- 战报页增加 WCL 趣味简报，首期收录 2026-07-10 的 Ntblack 战报和 2026-07-16 的同团活动战报，展示首领击杀、已匹配工会成员、整团/工会死亡王和趣味通关红度；每期保留 WCL 死亡页入口，并明确红度不代表装备掉落运气。
- 历史战报限制最近 12 条，长正文按需展开，避免页面持续变慢。
- 工会成员展示支持每位成员上传、调整裁切位置或撤下自己的共享图片，并自动滚动展示；文件选择时会立即拦截不支持或超过 8MB 的图片。
- 上传图片会在浏览器中压缩到 1.5MB 内；首页图片已压缩，非首页页面按需加载；论坛、战报和成员展示先显示公共内容，账号权限资料随后补齐，减少首屏等待并避免权限查询失败阻断公共页面。
- 生产构建会拆分 React、Supabase、图标和动画依赖，浏览器可长期缓存稳定文件，后续更新无需重复下载整站脚本。
- Cloudflare Pages 为带哈希的构建资源启用一年不可变缓存，并返回防嵌入、MIME 检查和隐私引用等浏览器安全响应头。
- Vite 构建会显式使用 React 生产模式，避免把开发调试代码发布到 Cloudflare Pages。
- 应用发生意外渲染错误时显示刷新和返回大厅入口；手机断网时显示明确提示，避免白屏或无反馈。
- 错误网址和失效分享链接显示明确的页面不存在状态，并提供大厅、活动和论坛出口，不再静默跳回首页。
- 网站提供键盘可见焦点、跳到主要内容入口和明确导航区域标签，不影响普通手机用户的界面。
- 活动、帖子和评论在服务层再次校验并清理输入，拒绝纯空格内容、无效时间和超过 40 人的异常活动。

升级论坛回复、成员展示和头像裁切位置时，在 Supabase SQL Editor 执行 `supabase/migrations/20260716_forum_showcase_and_crop.sql`。

执行 `supabase/migrations/20260717_forum_content_management.sql` 时，会在建立“每账号每活动一个名额”约束前清理历史重复报名：优先保留已确认记录，再保留最早的有效报名，避免旧数据导致迁移失败。

启用论坛分层回复、帖子编辑删除、活动发起人管理、单人单席位报名和战报删除权限时，执行 `supabase/migrations/20260717_forum_content_management.sql`。脚本可重复执行，也会补齐遗漏的评论回复字段。

启用“修改我的报名”时，在 Supabase SQL Editor 执行 `supabase/migrations/20260718_update_own_signup.sql`。该函数只允许成员修改自己报名中的角色、职责和备注，不允许更改活动、账号或报名确认状态，活动锁定后也会拒绝修改。
